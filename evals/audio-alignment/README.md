# Audio alignment eval: Whisper baseline vs ElevenLabs Forced Alignment

Технический эксперимент, **не миграция**. Сравнивает на одном и том же уже
готовом файле `public/audio/lesson-fr.mp3`, насколько точно подсветка идёт по
словам при двух источниках таймкодов:

1. текущий production-путь: `OpenAI TTS → Whisper → custom token mapping`
   (`src/data/lessonTimestamps.json`);
2. `тот же MP3 + точный исходный текст → ElevenLabs Forced Alignment`.

Ничего в приложении (bottom sheet, reader, `api/align-audio.ts`,
`lessonTimestamps.json`, TTS-провайдер/голос) не меняется. Результат — отчёт
+ страница, на которой можно на слух и визуально переключаться между двумя
вариантами во время одного воспроизведения.

## Как добавить ключ

1. Зайди на https://elevenlabs.io/app/settings/api-keys и создай API-ключ.
2. Скопируй `.env.example` в `.env` в корне проекта, если `.env` ещё нет.
3. Впиши ключ в строку `ELEVENLABS_API_KEY=` в `.env`. Файл `.env` в
   `.gitignore` — в репозиторий не попадёт.

Ключ нигде не логируется и не появляется ни в отчёте, ни в JSON, ни в
preview — все сгенерированные файлы содержат только таймкоды и текст.

## Как запустить

```bash
npm run eval:alignment
```

Первый запуск делает **один** платный запрос к ElevenLabs Forced Alignment и
сохраняет сырой ответ в `.cache/forced-alignment-raw.json` (гитигнорится).
Все последующие запуски используют этот кэш и сеть не трогают — так что
`npm run eval:alignment` можно перезапускать сколько угодно бесплатно после
первого раза (например, чтобы пересчитать отчёт после правки метрик).

Если нужно принудительно повторить именно платный запрос (например, изменился
исходный аудиофайл или текст):

```bash
npm run eval:alignment -- --fresh
```

Результат — `comparison.json`, `elevenlabs-timestamps.json`,
`whisper-timestamps.json`, `lesson-snapshot.json` и `report.md` в этой же
папке.

## Как открыть preview

```bash
npm run eval:alignment:preview
```

Выведет один локальный URL (например `http://localhost:4321`) — открой его в
браузере. Preview читает файлы, сгенерированные предыдущим шагом (`npm run
eval:alignment`) — если их ещё нет, страница скажет, что запустить сначала.

На странице: полный текст урока, тот же MP3, Play/Pause, скорость 0.8×/1×,
переключатель источника таймкодов (не перезапускает аудио — только меняет,
что используется для подсветки на следующем кадре), список 15 токенов с
наибольшей разницей (клик — переход и проигрывание с этого места), панель с
`startTime`/`endTime`/разницей выбранного слова, волнистое подчёркивание для
слов без таймкода в текущем источнике.

## Что платно, что нет

- Платно: ровно один вызов ElevenLabs Forced Alignment за прогон **без**
  `--fresh`-кэша (или каждый раз с `--fresh`).
- Бесплатно: сам preview (статический файловый сервер), повторные запуски
  `npm run eval:alignment` с существующим кэшем, чтение существующего
  Whisper-baseline (`src/data/lessonTimestamps.json`, только на чтение).

## Как вручную проверить результат

Метрики (coverage, разница в секундах, ElevenLabs `loss`) — это техническое
совмещение с сигналом, а не гарантия, что подсветка звучит естественно.
Обязательно прослушай preview на 1× и затем на 0.8×, переключая источник
несколько раз **не перезапуская** аудио, минимум в следующих местах:

- короткие слова (`de`, `à`, `et`, артикли, местоимения);
- слова с апострофом и с дефисом;
- начало и конец предложений, переход между абзацами;
- все 15 токенов из списка «наибольшая разница» на странице.

Подробный чек-лист признаков — в сгенерированном `report.md`, раздел «Токены
для ручной проверки», и раздел «Что нельзя определить автоматически».

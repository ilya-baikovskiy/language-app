# Утверждённая mobile-first UX-модель и правила интерфейса

> Статус: **актуальный источник истины для основного mobile UX**.
>
> Если старые примеры в других документах противоречат этому файлу или прототипу v8, использовать правила из этого файла. Прототип — визуальный референс, а не production-код для прямого копирования.

## 1. Продуктовая модель

Context Reader — не каталог курсов и не генератор уроков, который заставляет пользователя сначала настраивать тему. Основной сценарий:

```text
выбрать активный язык
→ быстро найти интересный материал
→ прочитать или прослушать
→ при необходимости открыть объяснение слова/фразы
→ сохранить полезную единицу
→ продолжить или повторить позже
```

Главная ценность — короткое интересное чтение на выбранном языке. Учебный план работает скрыто и не превращает интерфейс в курс.

## 2. Информационная архитектура

В мобильном приложении три основных раздела в нижней навигации:

1. **Выбрать** — рекомендации новых материалов.
2. **Мои тексты** — созданные, начатые и завершённые материалы активного языка.
3. **Учить** — сохранённые слова и фразы активного языка; активное повторение будет отдельным следующим этапом.

`Reader` не является четвёртой вкладкой. Он открывается из карточки или библиотеки как отдельный экран чтения.

Настройки остаются в верхней панели и не занимают отдельную нижнюю вкладку.

## 3. Нижняя стеклянная навигация

### Состав

```text
[ Выбрать ]   [ Мои тексты ]   [ Учить ]
```

У каждой вкладки всегда есть:

- простая контурная SVG-иконка;
- короткая подпись;
- touch target не меньше 44×44 px;
- `aria-current="page"` для активной вкладки.

### Визуальный стиль

- floating bar над нижним краем;
- учёт `env(safe-area-inset-bottom)`;
- умеренный blur и полупрозрачная светлая поверхность;
- тонкая светлая граница;
- мягкая тень;
- активная вкладка выделяется лёгкой внутренней подложкой, а не тяжёлой цветной кнопкой;
- ширина ограничена на планшете/desktop, но на телефоне занимает почти всю доступную ширину;
- меню не должно перекрывать последний контент: у страниц есть достаточный bottom padding.

### Reader

Во время чтения основное нижнее меню скрывается. Оно не должно конкурировать с:

- аудиоплеером;
- прогрессом чтения;
- переводом предложений;
- Bottom Sheet объяснения.

Возврат из Reader ведёт в тот раздел, из которого пользователь открыл материал.

## 4. Один глобальный активный язык

У приложения есть одно сквозное состояние:

```ts
activeLanguage: LanguageCode
```

Оно управляет всеми тремя основными разделами.

Пример:

```text
🇩🇪 Deutsch · A1
├── Выбрать: немецкие материалы
├── Мои тексты: немецкая библиотека
└── Учить: немецкие слова и повторение
```

### Переключатель

В верхней панели постоянно показана компактная кнопка:

```text
🇬🇷 Ελληνικά · A2⌄
```

Сейчас используются:

- 🇬🇷 Ελληνικά · A2
- 🇩🇪 Deutsch · A1
- 🇫🇷 Français · A1
- 🇬🇧 English · B2

Флаг используется только как быстрый визуальный маркер. Каноническая сущность в данных — код языка, а не страна/флаг. Выбор конкретного флага, особенно для English, можно пересмотреть позже.

В dropdown показываются флаг, название языка, уровень и короткая цель языка.

### Чего не делать

- не показывать локальные переключатели `Текущий язык / Все`;
- не смешивать материалы разных языков в основной библиотеке;
- не смешивать слова разных языков в основном разделе `Учить`;
- не добавлять флаги или `EL / DE / FR / EN` на каждую карточку;
- не переключать глобальный язык молча из-за открытия deep link на урок другого языка.

### Reader и deep links

Reader всегда использует язык и уровень, сохранённые в самом уроке. Если урок открыт внешней ссылкой и его язык отличается от `activeLanguage`, приложение не обязано автоматически менять глобальный контекст. Позже можно явно предложить перейти в соответствующий язык.

## 5. Экран «Выбрать»

### Верх страницы

Оставить:

- крупный заголовок **«Что почитаем?»**;
- глобальный переключатель языка в общей верхней панели;
- кнопку настроек;
- секцию **«Подобрано для тебя»**;
- вторичное действие **«Предложить другие»**.

Убрать:

- блок `Сегодня`;
- `5 идей · до 3 минут`;
- подзаголовок `Истории и короткие материалы для ...`;
- объяснения вроде `без генерации лишних уроков заранее`;
- каталог `Что вообще может генерироваться` с главного экрана;
- mastery, streaks, проценты плана и учебную терминологию.

### Подборка

Первая версия показывает пять карточек. Внутренние роли алгоритма не видны пользователю:

1. hero interest;
2. learning gap;
3. mandatory foundation;
4. diversity;
5. stretch/fresh.

Первая карточка крупнее остальных и занимает роль редакционной главной рекомендации. Отдельная чипса `Главная` не нужна: иерархия уже понятна по размеру.

## 6. Карточка материала

### Иерархия

Карточка содержит только:

1. изображение;
2. две чипсы поверх изображения;
3. крупный заголовок на русском;
4. одно русское описание;
5. кнопку `Читать`.

### Заголовок

В ленте и библиотеке заголовок показывается **на русском**, чтобы пользователь за долю секунды понял, интересна ли тема.

Не показывать под ним перевод на изучаемом языке. Этот эксперимент ухудшил быстрое сканирование ленты.

Пример:

```text
Почему в Швейцарии несколько языков
```

### Описание

Описание остаётся на русском. Учебная направленность объединяется с описанием в один абзац.

Допустимо:

```text
Простое объяснение того, почему язык зависит от региона. В тексте: страны, регионы и общение.
```

`В тексте: ...`:

- не оформляется отдельным блоком;
- не выделяется другим цветом;
- не получает заголовок `Что встретится`;
- читается как спокойное продолжение описания.

### Чипсы на изображении

Оставить ровно две:

1. provenance:
   - `AI-история`;
   - `На основе фактов`;
   - `На основе источников`;
2. длительность:
   - `2 мин`;
   - `3 мин`.

Не дублировать над заголовком:

- страну;
- длительность;
- уровень;
- формат;
- язык.

Не показывать одновременно дублирующие метки вроде:

- `Спокойный рассказ`;
- `Художественный материал`;
- `Главная рекомендация`.

### CTA

Основной текст кнопки — `Читать`. Выбор запускает создание полного Lesson только после нажатия.

## 7. Изображения

Изображения на карточках остаются важной частью продукта: они помогают эмоционально выбирать материал и отличают ленту от учебного списка.

Целевое решение — генерировать изображения через AI.

До реализации нужно отдельно определить единую visual direction. Возможные правила:

- единый спокойный editorial/illustrative стиль;
- одинаковая степень реализма;
- одинаковая композиционная плотность;
- место для читаемых чипсов;
- отсутствие текста, логотипов и интерфейсных элементов внутри картинки;
- factual и fiction могут различаться только системно, а не случайным набором стилей.

До AI image pipeline допустимы нейтральные placeholders. Не копировать декоративные символы прототипа как финальные изображения.

## 8. Темы, страны и языки — независимые измерения

### Глобальные темы

Темы общие для всех изучаемых языков:

- Повседневная жизнь;
- Культура;
- История;
- Места и архитектура;
- Язык и выражения;
- Еда;
- Путешествия и природа;
- AI и технологии;
- Общество и политика;
- Свежие события.

### Глобальные страны и регионы

Страны и регионы тоже общие:

- Греция;
- греческие острова;
- Кипр — преимущественно история и культура;
- Германия;
- Бавария;
- Австрия;
- Швейцария;
- Франция;
- Бельгия;
- Люксембург;
- франкоязычные регионы;
- другие страны и регионы.

### Главное правило

Язык чтения не обязан совпадать со страной материала.

Пример:

```text
activeLanguage: Ελληνικά A2
subject: Почему в Швейцарии несколько языков
lesson: адаптация на греческом A2
```

Одна контентная идея может быть развернута в:

- Greek A2;
- German A1;
- French A1;
- English B2.

Это не дословные переводы. Каждый Lesson заново адаптируется под язык, уровень и учебную задачу.

### Настройки тем

Темы и страны находятся в настройках, а не в основной ленте.

Изменение настроек:

- влияет на будущие рекомендации;
- может сразу пересобрать текущую подборку;
- не удаляет уже созданные уроки;
- не отменяет скрытый учебный план;
- не должно позволять оставить полностью пустой набор без понятного fallback.

Скрытый учебный план может покрывать базовую лексику через любую разрешённую пользователем оболочку.

## 9. Экран «Мои тексты»

Экран показывает только уроки `activeLanguage`.

Состав:

1. заголовок `Мои тексты`;
2. блок `Продолжить`, если есть незавершённый урок;
3. список/сетка всех материалов активного языка;
4. статусы и прогресс.

### Продолжить

Выбирать урок по `lastOpenedAt`, а не по максимальному проценту прочитанного.

Карточка показывает:

- русский заголовок;
- уровень урока на момент создания;
- длительность;
- прогресс;
- CTA `Продолжить`.

### Состояния

```text
creating
ready
started
completed
failed
```

Урок хранит собственные `language` и `level` snapshot. Изменение уровня в настройках не переписывает старые уроки.

Один и тот же canonical subject может иметь разные уроки на разных языках. Нельзя идентифицировать урок только заголовком; нужен стабильный `lessonId`.

## 10. Экран «Учить»

Экран показывает только сохранённые единицы `activeLanguage`.

Основные элементы:

- очередь `На сегодня`;
- количество слов и фраз для повторения;
- CTA `Начать`;
- список сохранённого;
- target-language форма;
- русский смысл;
- источник;
- кнопка произношения.

Не показывать `Все языки` в основном UX.

Полноценная SRS-модель, интервалы, mastered state и карточная тренировка не входят в текущий content-feed MVP. Экран можно реализовать как подготовленную оболочку, не имитируя готовый алгоритм.

## 11. Reader

Reader открывается после готовности Lesson.

### Заголовок

Внутри Reader порядок другой, чем в ленте:

1. заголовок на языке урока;
2. небольшой русский перевод;
3. level + duration;
4. текст на изучаемом языке.

Это допустимо, потому что пользователь уже выбрал тему и перешёл к реальному чтению.

### Остальные утверждённые принципы

- tap по слову/фразе открывает contextual Bottom Sheet;
- первое состояние Bottom Sheet — смысл в контексте и простой разбор;
- подробная грамматика скрыта в `Подробнее`;
- перевод предложений — отдельный режим;
- сохранение слова/фразы — отдельное действие, не эквивалент tap;
- TTS и karaoke timing сохраняются;
- глобальный переключатель языка внутри Reader не показывается;
- progress сохраняется автоматически.

## 12. Настройки

В модальном экране настроек:

1. уровни всех языков;
2. общие темы;
3. общие страны и регионы.

Текущие уровни прототипа:

- Greek A2;
- German A1;
- French A1;
- English B2.

Уровень хранится отдельно для каждого языка. Темы и страны — общие пользовательские предпочтения.

## 13. Генерация

После нажатия `Читать`:

1. использовать `activeLanguage` и его текущий selected level;
2. создать language-specific `LessonBlueprint` из canonical content idea;
3. запустить существующий Lesson pipeline;
4. сразу добавить запись в библиотеку активного языка со статусом `creating`;
5. показать реальные стадии;
6. после готовности открыть Reader или дать продолжить из библиотеки.

Один выбор не должен создавать дубликаты при retry/reload.

## 14. Модель данных, следующая из UX

Минимальные состояния:

```ts
interface AppPreferences {
  activeLanguage: LanguageCode;
  enabledTopicIds: string[];
  enabledCountryOrRegionIds: string[];
}

interface LanguageProfile {
  language: LanguageCode;
  selectedLevel: CEFRLevel;
  targetLevel?: CEFRLevel;
  goalType: LanguageGoalType;
}
```

Canonical content idea не принадлежит одному языку. Feed item добавляет контекст:

```ts
interface FeedItem {
  cardId: string;
  language: LanguageCode;
  targetLevel: CEFRLevel;
  position: number;
  slot: FeedSlot;
}
```

Урок обязательно хранит:

- `lessonId`;
- `cardId` / canonical subject id;
- `language`;
- `level` snapshot;
- `lastOpenedAt`;
- `progress`;
- `status`;
- artifact refs.

Saved learning unit хранит `lessonId` и `language`.

## 15. Трекинг, необходимый для проверки модели

Минимальные UX-события:

- `bottom_navigation_selected`;
- `global_language_changed`;
- `settings_opened`;
- `topic_preferences_changed`;
- `country_preferences_changed`;
- `feed_viewed`;
- `feed_refreshed`;
- `card_impression`;
- `card_opened`;
- generation events;
- `lesson_opened` с entry point;
- progress/completion;
- word/phrase taps;
- saves;
- translation toggle;
- feedback.

Проверять отдельно:

- как часто язык переключают за сессию;
- возвращается ли пользователь к последнему активному языку;
- открывает ли `Продолжить`;
- насколько русские заголовки повышают скорость выбора;
- становится ли `Учить` частотным после накопления сохранений.

## 16. Accessibility и mobile acceptance

- semantic buttons/links;
- visible focus;
- `aria-expanded` и listbox semantics у language selector;
- reduced motion;
- контраст текста и стеклянных поверхностей;
- важный текст не находится внутри изображения;
- touch targets ≥ 44 px;
- bottom nav и modals учитывают safe area;
- sticky/floating элементы не перекрывают контент;
- интерфейс тестируется минимум на узком iPhone viewport и desktop fallback.

## 17. Что пока намеренно не решено

- финальный стиль AI-изображений;
- реальный SRS и тренировка;
- нужна ли когда-либо смешанная библиотека всех языков;
- deep-link UX для урока другого языка;
- выбор флага для English;
- поведение bottom nav при интенсивном scroll;
- финальная типографика и production design tokens;
- постоянная база данных.

Эти вопросы не должны блокировать первую реализацию утверждённого основного пути.

## 18. Acceptance criteria для текущего UX

- на всех трёх вкладках один и тот же активный язык;
- смена языка обновляет feed, library и learn;
- нет `Текущий язык / Все`;
- в feed/library русские заголовки;
- в Reader заголовок на языке урока + русский перевод;
- карточка имеет только две чипсы на изображении;
- hero не имеет метки `Главная`;
- страна и минуты не дублируются рядом с заголовком;
- темы/страны отсутствуют на main feed и доступны в settings;
- темы/страны работают для любого языка;
- bottom nav состоит из трёх утверждённых разделов;
- Reader скрывает bottom nav;
- prototype v8 проходит основные кликабельные сценарии.

# Context Reader — план реализации

MVP-прототип интерактивного языкового ридера: React + TypeScript, без backend и API-ключей,
один подготовленный французский текст (A2–B1), синхронная озвучка через Web Speech API,
кликабельные слова/фразы, Bottom Sheet с контекстным объяснением.

Полное ТЗ — см. исходный документ пользователя. Ниже — рабочий план и договорённости.

## Договорённости

- Работаем **поэтапно**, с чекпоинтом (демо в браузере) после каждого этапа — без
  автономного прогона всех 5 стадий разом.
- Текст урока и разметку аннотаций пишу сам (французский A2–B1, ~4 мин чтения).
- Контент закладывается архитектурно так, чтобы позже заменить статический
  `sampleLesson.ts` на генерацию через OpenAI API (пользователь присылает статью/ссылку/
  тему → сервис генерит текст и разметку по нашим гайдам). Сейчас: только локальный
  провайдер, но за интерфейсом, а не захардкожен в компонентах.
- Проект — полноценный Vite + React + TS репозиторий с `git init`, а не одностраничный
  artifact.

## Этапы реализации

### Этап 0 — Инициализация
- Vite + React + TS скаффолд.
- `git init` + первый коммит.
- Структура папок (components / hooks / services / data / types / styles).
- CSS-переменные дизайн-токенов.

**Чекпоинт:** пустой проект запускается (`npm run dev`).

### Этап 1 — Статический интерфейс
- Header, центральная колонка со статьёй (mock-текст), статичный player, статичный
  Bottom Sheet.
- Адаптивная вёрстка mobile/desktop без логики.

**Чекпоинт:** визуальный обзор в браузере — **этот этап сейчас обсуждаем отдельно**.

### Этап 2 — Интерактивные токены
- Разбивка текста на sentences/tokens.
- Клик по слову → резолвинг аннотации (слово vs фраза).
- Два визуальных состояния подсветки (произносимое слово / изучаемая фраза).
- Открытие/закрытие Bottom Sheet без скачков скролла.

### Этап 3 — Озвучивание
- `NarrationAdapter` интерфейс + `BrowserSpeechAdapter` (Web Speech API) + fallback
  приблизительной синхронизации.
- Play / Pause / Stop, 5 скоростей (0.6×–1.5×), старт с любого слова, «Продолжить отсюда».
- Тестируется вживую в браузере (Web Speech API нельзя проверить иначе).

### Этап 4 — Контент и объяснения
- Французский текст A2–B1 + JSON-разметка аннотаций (леммы, грамматика,
  contextualMeaning на русском, примеры) через отделённый content-слой.
- Наполнение блоков Bottom Sheet: значение в контексте, краткая/расширенная грамматика,
  примеры, кнопки прослушивания, заглушка «Спросить AI».

### Этап 5 — Полировка и приёмка
- Мобильная/десктопная проверка, клавиатура (Space/Escape), доступность, reduced-motion,
  fallback при недоступном TTS или отсутствующей аннотации.
- Финальная сверка с чек-листом приёмки (26 пунктов из ТЗ).

## Структура страницы (для обсуждения визуала — Этап 1)

**Четыре области:**
1. **Верхняя панель** — кнопка возврата, название материала, язык · уровень · длительность,
   кнопка настроек текста.
2. **Область материала** — центральная колонка (desktop max 680–760px, tablet ~80%,
   mobile — экран минус 20–24px с боков). Крупный шрифт (21–23px desktop / 19–21px mobile),
   line-height 1.65–1.8, интервал между абзацами 1.2–1.5 строки. Текст выглядит как обычная
   статья — никакой постоянной разметки слов.
3. **Панель озвучивания** — компактный sticky-player снизу, появляется после старта чтения:
   Play/Pause, Stop/Reset, текущая скорость, индикатор прогресса, переход к настройкам
   скорости.
4. **Bottom Sheet** — снизу по клику на слово. Мобильный: 55–85% высоты экрана. Десктоп:
   снизу по центру, ограниченная ширина, скруглённые верхние углы, лёгкое затемнение фона.

**Состояния интерфейса:** Idle → Playing → Paused → Inspecting (клик по слову) → Stopped →
Completed → Error (TTS недоступен). До первого взаимодействия — ничего «учебного»: без
цветовой маркировки, подчёркиваний, боковых панелей, переключателей режимов.

**Визуальный стиль:**
- Минимализм, editorial-дизайн, крупная типографика, много воздуха.
- Подсветка текущего слова — мягкий фон accent-цвета низкой непрозрачности,
  переход 100–150ms.
- Изучаемая фраза — более заметное выделение (фон + тонкое подчёркивание), не похожее на
  стандартное браузерное выделение текста.
- CSS-переменные из ТЗ (можно менять):
  `--background #fff · --surface #f7f7f8 · --text-primary #18181b ·
  --text-secondary #71717a · --accent #6d5dfc · --accent-soft rgba(109,93,252,.14) ·
  --border rgba(24,24,27,.1) · --overlay rgba(0,0,0,.24)`
- Анимации: Bottom Sheet 220–300ms, подсветка слова 100–150ms, грамматика 180–250ms,
  появление player 180–250ms. Учитывать `prefers-reduced-motion`.

## Открытые вопросы к визуалу (обсудить перед Этапом 1)

- Общая эстетика: ближе к Kindle/книге, к Medium-статье, или к чему-то более «продуктовому»
  (Duolingo-lite, но без геймификации)?
- Тема по умолчанию: светлая, тёмная, или сразу обе с переключателем в настройках?
- Шрифты: системный стек или конкретная пара (serif для текста статьи + sans для UI)?
- Bottom Sheet на десктопе — модальный оверлей на весь экран по ширине или строго
  привязанный к ширине колонки текста?
- Нужен ли какой-то «обложечный» визуальный элемент на первом экране (пример из ТЗ:
  заголовок + язык/уровень/длительность) или чисто текстовый header без изображений?

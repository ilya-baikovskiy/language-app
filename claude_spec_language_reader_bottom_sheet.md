# Bottom Sheet — ответы на комменты из BOTTOM_SHEET_PROPOSAL.md

Продолжение того разбора — два твоих вопроса оказались не абстрактными, я проверил их
против реального кода `ExplanationSheet.tsx`/`GrammarSummary.tsx`/`ExampleList.tsx`, и оба
указывают на настоящие пробелы. Отвечаю по каждому предметно.

## 1. «А если я только учусь и название грамматики мне ничего не говорит?»

Проверил порядок рендера в `ExplanationSheet.tsx` — вот что сейчас на самом деле происходит:

```
[header: слово]
[теги: partOfSpeech, grammarLabel]   ← например «passé composé» — голый, без пояснения
[короткий перевод]
──────────────
[В этом предложении: contextualMeaning]
[Конструкция: constructionExplanation]  ← может быть длинным
──────────────
[Грамматика: grammarSummary]            ← вот тут наконец простое объяснение термина
[Детали грамматики] [Другие значения]
[Примеры]
```

`grammarLabel` — это осознанно голый технический ярлык (по промпту он остаётся на языке
текста, не адаптируется). Но его расшифровка простыми словами (`grammarSummary`, специально
спроектирован как «1-2 предложения, без учебникового жаргона» — это уже заложено в промпте
генерации) рендерится в самом низу, ПОСЛЕ контекстного значения и конструкции. Между тегом
и его объяснением может быть несколько абзацев текста. Для новичка это ровно та проблема,
которую ты описал: видишь «passé composé» сразу, а что это значит — только долистав.

**Правка**: переставить `grammarSummary` так, чтобы он шёл сразу за тегами, а не в самом
низу — тег и его расшифровка простыми словами должны быть визуально одним блоком, без
контента между ними. Технически: рендерить короткую версию grammarSummary прямо в верхнем
блоке (там же, где сейчас голые теги), а не как отдельную секцию внизу. Сама секция
«Грамматика» внизу остаётся для тех случаев, когда есть что сказать ещё (grammarDetails,
другие значения) — но первое, самое важное пояснение не должно требовать пролистывания.

Это не отменяет три уровня раскрытия из основного предложения — просто уточняет, что
«тег + его расшифровка» это неразделимая пара на уровне 1, а не тег на уровне 1 и
расшифровка где-то в уровне 2/3.

## 2. «Пример контекста на русском — как сравнивать? Может, на языке текста?»

Тоже проверил — и это реальный пробел, не недопонимание. `selection.sentenceText`
(само французское предложение) в компонент действительно передаётся, но сейчас
используется ТОЛЬКО как аргумент для кнопки озвучки (`onSpeak(selection.sentenceText)`).
Само предложение нигде не отображается текстом внутри шита — видна только русская
`contextualMeaning`. Чтобы увидеть французский текст, нужно посмотреть вверх, на статью —
а раз шит открыт поверх неё (или занимает нижнюю часть экрана), это неудобно и это
ровно тот UX-разрыв, который ты поймал: русское объяснение висит без французского якоря
рядом, сравнивать не с чем.

(Примеры `ExampleList.tsx` — там всё уже двуязычно, `targetText` (франц.) + `translation`
(рус.) рядом, это ок и вопроса не касается — я специально перепроверил, чтобы не путать
два разных места.)

**Правка**: в секции «В этом предложении» показывать само французское предложение
(с выделением/подсветкой смыслового фрагмента — слова или фразы, к которой относится
объяснение) НАД русским текстом contextualMeaning, а не только через отдельную кнопку
озвучки. Целевой фрагмент можно выделить жирным внутри sentenceText через простой
substring-match по `displayText` (они из одного источника, должны совпадать почти всегда;
редкие расхождения по регистру/элизии — приемлемая погрешность, не стоит городить сложный
токен-левел рендеринг ради этого).

## Как это меняет исходное предложение

Обновлённая структура верхней части шита (только то, что меняется):

```
[header: слово 🔊]
[теги + их расшифровка простыми словами — одним блоком, без контента между ними]
[короткий перевод]
──────────────
[Французское предложение с выделенным фрагментом]   ← новое
[В этом предложении: contextualMeaning (рус.)]
[Конструкция, если есть]
──────────────
[остальное — детали грамматики, другие значения, примеры — как в исходном предложении]
```

Кросс-подсветку леммы по всей статье (пункт 3 из BOTTOM_SHEET_PROPOSAL.md) это не
затрагивает — там уже принято решение не делать, я его не пересматриваю.

# CLAUDE.md

Context Reader — интерактивный французский языковой ридер (React 19 + TypeScript + Vite,
без backend). Полное ТЗ и план — `PLAN.md`; визуальные решения — `DESIGN.md`; архитектура
AI-пайплайна (озвучка, генерация аннотаций) — `AI_PIPELINE.md`.

## Начало сессии

Прочитай **`PROGRESS.md`** первым делом — это живой статус проекта (что сделано, что в
процессе, что дальше, открытые вопросы). Он коммитится в git специально для того, чтобы
быть доступным при работе с другой машины или из claude.ai/code, а не только там, где
изначально шла разработка.

## Рабочие договорённости (обязательные, не только предпочтения)

- Работаем **поэтапно** (Этапы 0–5 из `PLAN.md`), с чекпоинтом-демо после каждого этапа —
  не прогонять несколько этапов подряд без остановки.
- **После завершения каждого этапа или заметного куска фронтенд-работы — проактивно
  запускай `npm run dev`**, не дожидаясь просьбы, чтобы результат сразу можно было
  посмотреть в браузере.
- Контент и провайдеры (TTS, генерация текста) закладываются за интерфейсами
  (см. `src/services/narration/NarrationAdapter.ts`), не хардкодятся в компоненты.
- Backend/генерация новых уроков — сознательно отложены до Этапа 5 и явного сигнала
  пользователя (см. `AI_PIPELINE.md`). Не начинать реализовывать без прямого запроса.
- По завершении этапа или значимого решения — обнови `PROGRESS.md` (и, если это архитектурное
  или визуальное решение, `AI_PIPELINE.md`/`DESIGN.md`).

## Параллельная работа через субагентов

В `.claude/agents/` определены субагенты под реальные потоки работы этого проекта —
вызывай их через Agent tool, когда задача явно попадает в один из потоков, или когда
нужно вести несколько независимых задач параллельно:

- **frontend** — компоненты/хуки/CSS в `src/` по `PLAN.md`/`DESIGN.md`.
- **content-writer** — французский текст урока и русскоязычные аннотации
  (`src/data/`, схема в `src/types/lesson.ts`, правила в `AI_PIPELINE.md`).
- **audio-pipeline** — TTS-скрипты, генерация озвучки и таймкодов (`scripts/`,
  `public/audio/`, `src/data/lessonTimestamps.json`).
- **reviewer** — read-only проверка по DESIGN.md/PLAN.md/доступности перед коммитом.

Если запускаешь несколько агентов **параллельно, которые будут редактировать файлы**,
запускай их с `isolation: "worktree"` — иначе они будут писать в одно и то же рабочее
дерево и могут затирать правки друг друга. Чисто read-only агентов (`reviewer`,
исследование) это не касается.

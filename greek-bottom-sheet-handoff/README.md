# Greek reader bottom sheet — handoff package

Этот пакет фиксирует согласованную логику и визуальные правила объяснения слова или фразы в reader.

## Что внутри

- `BOTTOM_SHEET_HANDOFF.md` — основной продуктовый и интерфейсный контракт.
- `AI_IMPLEMENTATION_PROMPT.md` — готовое задание для Claude Code / Codex.
- `content-example.json` — пример разделения короткого и подробного ответа AI.
- `screenshots/` — шесть снимков актуальной рабочей версии.

## Как передать в основной проект

1. Положить всю папку в репозиторий, например:

   `docs/features/reader-word-explanation/`

2. Открыть проект в Claude Code или Codex и отправить содержимое файла `AI_IMPLEMENTATION_PROMPT.md`.

3. Попросить AI сначала изучить существующие:

   - reader и разметку кликабельного текста;
   - bottom sheet / modal primitives;
   - дизайн-токены, типографику и иконки;
   - текущий AI/API-пайплайн перевода и объяснения слов.

4. Переносить не демонстрационную страницу целиком, а правила и состояния из handoff. Существующая дизайн-система проекта имеет приоритет; скриншоты задают иерархию, плотность и поведение.

5. После реализации проверить сценарии из раздела «Критерии приёмки» в `BOTTOM_SHEET_HANDOFF.md` и приложить мобильные скриншоты результата.

## Источник истины

Для последней итерации источником истины являются этот handoff и приложенные скриншоты. Опубликованный ранее прототип может отставать на несколько последних правок.

## Проблема, которую решаем

Сейчас в шите все поля аннотации всегда полностью развёрнуты: перевод → «в этом
предложении» → конструкция → грамматика (summary + details) → все примеры. Для тривиального
слова («de», «un») это много визуального шума ради одной строки перевода. Для богатого случая
(неправильный глагол, идиома) — тот же макет уезжает в скролл.

Второй, не связанный напрямую фактор: в параллельной задаче объяснение слова теперь будет
подгружаться лениво (по клику, не заранее для всего урока) — значит, в шите появится
промежуточное состояние загрузки, и это тоже нужно учесть в иерархии.

## Предложение: три уровня раскрытия

```
┌─────────────────────────────────────┐
│  ▬ handle                            │
│  mot  🔊                        [×]  │  ← Уровень 1: всегда видно
│  /pronunciation/                     │
│  от lemma · POS · grammar-tag        │
│  Короткий перевод.                   │
│  ───────────────────────────────     │
│  В этом предложении  🔊              │  ← Уровень 2: всегда видно,
│  Контекстное значение — грамматика   │    одним текстовым потоком
│  и конструкция, если есть, продол-   │    (без отдельных подзаголов-
│  жают тот же абзац, без своих        │    ков-блоков, как сейчас)
│  подзаголовков и рамок.              │
│  ───────────────────────────────     │
│  ▸ Подробнее про грамматику          │  ← Уровень 3: свёрнуто по
│  ▸ Другие значения                   │    умолчанию (<details>, как
│  Пример: «...» — перевод             │    уже сделано для деталей
│     ▸ +2 примера                     │    грамматики/значений —
│                                       │    расширяем тот же приём
│  [Продолжить отсюда] [✦ Спросить AI] │    и на примеры)
└─────────────────────────────────────┘
```

Для простых слов (0-1 пример, нет grammarDetails/otherMeanings) уровень 3 либо не рендерится
вообще, либо занимает одну строку — визуально ничего не меняется по сравнению с сегодняшним
днём. Меняется поведение только для «богатых» карточек.

## Состояние загрузки (ленивая генерация)

Пока объяснение ещё не подгружено с сервера, тумблеры уровня 3 («▸ Подробнее», «▸ Другие
значения») не должны рендериться вовсе — иначе можно показать пустой тумблер, который после
загрузки окажется ни на что не разворачивающимся. Скелетон/лоадер покрывает только
всегда-видимую часть (слово, перевод, «в этом предложении»).

> КОММЕНТ:

## Кросс-подсветка той же леммы в тексте

**Рекомендация ресёрч-агента: не делать.**

- `DESIGN.md` прямо называет антипаттерном постоянную цветовую разметку слов (LingQ) —
  «превращает статью в учебник». Даже временная, sheet-scoped подсветка добавляет ещё один
  слой ambient-состояния поверх уже существующих (текущее слово при чтении, инспектируемая
  фраза, выбранный токен) — это против принятого направления «спокойный editorial, не игровой
  UI».
- Ценность тоже под вопросом: тексты в приложении одноразовые (один урок читается один раз
  подряд), повторы одного слова в видимом абзаце — редкость.
- Если ценность всё же нужна — предлагается не ambient-подсветка по странице, а явная строка
  внутри самого шита: «Встречается ещё 2 раза →» с переходом/скроллом к следующему вхождению.
  Тот же эффект («видно, что слово повторяется»), без визуального шума на странице.

# QA_TESTING — ручной и полуавтоматический прогон Content System v1.2

Этот файл — чеклист для проверки всего, что было построено в PR 1–4
(`docs/content-system-v1.2/`) плюс существующий reader-пайплайн (генерация
текста, озвучка, Bottom Sheet), который эти PR используют, не переписывая.

## Как это устроено

- Разделы 1–8 — ручные шаги в браузере. Пробовал прогнать их автоматически
  через Playwright (`scripts/qa-playwright-check.mjs`, реальный Chromium,
  предустановленный в этой песочнице) — не получилось: egress-прокси этой
  сессии пускает обычные HTTP-запросы (`curl`/`fetch`) на preview-хост, но
  блокирует туннель для полноценного браузерного трафика на тот же хост
  (`CONNECT` → `403`, это осознанная политика организации, не флейк и не
  баг флагов Chromium — см. git-историю коммитов на этой ветке). Так что
  визуал, тайминг аудио и реальные клики — только вручную, человеком.
- Раздел 9 (матрица частей речи) — прогнан напрямую через API
  (`/api/generate-text`, `/api/generate-annotation`) на реальном
  preview-деплое, без браузера. Это проверяет **контент**, который AI выдаёт
  для разных частей речи — то, что вы увидите в Bottom Sheet, если откроете
  тот же токен в браузере, должно совпадать с тем, что здесь записано.
  Подробности реального прогона и находки — см. раздел 9 целиком.
- **Не дёргать в ходе QA** (пишут в общий с продакшеном Blob store):
  `/api/save-lesson`, `/api/lesson-status`, `/api/app-preferences`,
  `/api/language-profiles`, `/api/events-batch`. Разделы 1–8 просто открывают
  их через UI как обычный пользователь — это нормально, там нет прямых вызовов.
- Ссылка для собственного тестирования: `<PREVIEW_URL>` (актуальный deployment
  ветки `content-system-docs-v1.2` — если открылась заглушка Vercel SSO,
  залогиньтесь в свой аккаунт, у вас уже есть доступ к проекту).

## 1. Лента «Выбрать»

- [ ] Открыть приложение → таб «Выбрать» активен по умолчанию.
- [ ] Заголовок «Что почитаем?», секция «Подобрано для тебя», ровно 5 карточек.
- [ ] Первая карточка (hero) заметно крупнее остальных, **без** бейджа «Главная».
- [ ] На каждой карточке ровно 2 чипсы поверх изображения: provenance
      (`AI-история`/`На основе фактов`/`На основе источников`) + длительность
      (`N мин`). Никаких стран/уровней/языка/формата на карточке.
- [ ] «Предложить другие» → набор карточек меняется (может частично
      повторяться, если seed-пул исчерпан — это ожидаемо, см. `feed.ts`).
- [ ] Сменить язык в TopBar (dropdown вверху) → лента перезагружается под
      новый язык, набор карточек другой.
- [ ] Открыть настройки (шестерёнка) → изменить уровень текущего языка → лента
      обновляется под новый уровень (карточки без подходящего уровня пропадают).
- [ ] Включить/выключить тему или страну в настройках → лента реагирует
      (пустая лента при полностью выключенных фильтрах не должна происходить —
      если увидите пустой экран без объяснения, это баг).

## 2. Card → Lesson генерация

- [ ] Клик «Читать» на любой карточке → открывается экран прогресса с
      реальными стадиями (`Готовим план` → `Пишем текст` → `Озвучиваем` →
      `Синхронизируем аудио` → `Сохраняем`), не мгновенный переход.
- [ ] По готовности — автоматический переход в Reader с текстом на выбранном
      языке и связанной темой карточки.
- [ ] Вернуться в «Мои тексты» → урок появился там со статусом `ready`
      (открывается кликом).
- [ ] Повторно кликнуть «Читать» на **той же** карточке при том же языке/
      уровне → должен открыться уже готовый урок без повторной генерации
      (не создаётся вторая запись в библиотеке — проверить, что в «Мои
      тексты» не появился дубликат).
- [ ] Если генерация упадёт (например, временная сетевая ошибка) — в
      библиотеке должна появиться запись со статусом `failed` и кнопкой
      «Повторить», которая перезапускает генерацию в тот же урок.

## 3. Библиотека («Мои тексты»)

- [ ] Показывает только уроки activeLanguage (сменить язык в TopBar → список
      меняется).
- [ ] `creating`-запись отображается неактивной с пометкой «Готовится…», не
      кликабельна как обычная карточка.
- [ ] `failed`-запись — с пометкой «Не удалось сгенерировать» и «Повторить»
      (если у записи есть `cardId`; у уроков из ручной генерации кнопки не
      будет — это ожидаемо).
- [ ] «+ Новый урок» (ручная генерация по своей теме/тексту) по-прежнему
      работает независимо от карточек — выбор языка/уровня/объёма/провайдера
      озвучки, результат открывается в Reader и появляется в библиотеке.

> Не прогнано автоматически в этой сессии — намеренно, чтобы не тратить
> лишний реальный вызов OpenAI+ElevenLabs (см. решение пользователя:
> тестируем на одной карточке, не на пяти). `GenerateLessonPage` вызывает тот
> же `generateLesson()` (см. `generateLessonPipeline.ts`), что и card-flow —
> единственная разница в сборке `InputSource` (свободная тема/текст вместо
> `LessonBlueprint`). `/api/generate-text` и `/api/generate-annotation`
> подтверждённо работают на Preview с реальными ключами (раздел 9) — но сам
> клик «Читать»/«Сгенерировать» через UI (раздел 2 целиком, включая этот
> пункт) остаётся непроверенным в этой сессии (браузер заблокирован, см.
> «Как это устроено» выше) — оба пункта нужно пройти вручную.

## 5. Bottom Sheet — детали слова (тир 1 и тир 2)

- [ ] Тап по любому слову → снизу выезжает Sheet с: часть речи, форма слова
      как она стоит в тексте, перевод в этом контексте, при наличии — строка
      «связанная фраза» (source → translation с ярлыком).
- [ ] Тап по соседнему слову внутри той же фразы → **другое** объяснение
      (не то же самое, что для первого слова — это ключевой критерий приёмки
      Bottom Sheet v2, ломался в старой архитектуре с фразовой разметкой).
- [ ] «Подробнее» → подгружается тир 2: секции explanation/table/
      bilingualPairs/grammarNote. В таблицах **нет** заголовков колонок и
      **нет** подсветки текущей формы слова ни в одной строке.
- [ ] Клик по слову с пунктуацией (запятая/точка рядом) не ломает выделение
      и не включает пунктуацию в объяснение.

## 6. Произношение слова/фразы в Sheet

- [ ] Кнопка-динамик в Sheet проигрывает клип именно этого слова/фразы (короткий

- [ ] Повторный клик по тому же слову — клип проигрывается снова без ошибок

## 7. Перевод предложений

## 8. Сохранение слов

**Итог: 20/23 полностью соответствуют документированным правилам, 3 реальные находки:**
1. **Опечатка** «пассé composé» (смешение кириллицы/латиницы) — косметическая, невысокий приоритет.
2. **Токенизация** инвертированного вопроса («As-tu» как один токен) корректно объясняется существующей логикой токенайзера (дефис = связка), но прямой перевод в тир 1 подмешивает смысл соседнего токена — стоит перепроверить на ещё нескольких примерах инверсии, прежде чем менять промпт.
3. **Баг**: `partOfSpeech` для относительного местоимения «qui» ошибочно возвращается как «союз», хотя тир 2 в том же ответе верно называет его местоимением — рассинхрон между тир 1 и тир 2 классификацией, стоит завести отдельно как баг в `generateAnnotations.ts`/промпте.

Остальное — качественный, грамматически точный контент: hint показывается
именно там, где это оправдано (`se lève`, согласование причастия, предлоги в
устойчивых сочетаниях, указательные местоимения), и не показывается для
самодостаточных слов; 4-частная структура для глаголов соблюдена; таблицы
без заголовков, 3 колонки; функциональные слова и имена собственные получают
минимальную (2-3 секции или меньше) обработку, как и предписано.

### Как воспроизвести

```bash
BASE=<preview-url>
BYPASS=<vercel-protection-bypass-token>
curl -sS -X POST "$BASE/api/generate-text" -H "Content-Type: application/json" \
  -H "x-vercel-protection-bypass: $BYPASS" \
  -d '{"input":{"kind":"topic","prompt":"<тема с богатой грамматикой>"},"level":"B1","words":180,"language":"fr"}' \
  -o lesson.json

npx tsx scripts/qa-tokenize-sample.mjs lesson.json lesson-tokenized.json fr-FR

# затем для каждого выбранного token id — POST на $BASE/api/generate-annotation
# с {target:{tokenId, sentence}, level, tier: 'basic'|'details', language}
# (см. NODE_USE_ENV_PROXY=1, если запускать из этой же песочницы — plain
# fetch() не читает $HTTPS_PROXY по умолчанию, curl — читает)
```
